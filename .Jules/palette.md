## 2026-06-17 - README UX for Doc-Only Repos
**Learning:** When the primary HTML UI files are absent from the current repository state, enhancing the documentation's UX (e.g., adding a clickable Table of Contents, emojis for scannability, or '[↑ Back to Top](#table-of-contents)' links) is the only valid micro-UX scope.
**Action:** Always check the repository root for UI source files before proposing application-level UX changes; if only README.md exists, focus on documentation-centric micro-UX improvements.
