## 2025-05-14 - Blockchain Submission Feedback
**Learning:** Users submitting blockchain transactions need immediate visual feedback (e.g., "Submitting...") and a disabled button state to prevent double-spending or redundant signatures, especially given the perceived latency of decentralized networks.
**Action:** Implement a "Submitting..." state on submission buttons that disables the button and provides clear text feedback while the transaction is being processed.
