## 2025-05-14 - Functional and Accessible Upload Patterns
**Learning:** Transforming static upload placeholders into functional components requires balancing keyboard accessibility with visual feedback. A `div` used as a button needs `tabindex="0"`, `role="button"`, and explicit `keydown` listeners for 'Enter' and 'Space' to trigger a hidden `<input type="file">`. Additionally, providing clear visual feedback during drag-and-drop (e.g., border color changes) and safely rendering selected filenames (using `textContent`) significantly improves the "loop" of user interaction.
**Action:** When implementing custom upload areas, always pair the interactive container with a hidden input, ensure keyboard triggers are mapped, and use Tailwind's `focus` utilities to make the focus state unmistakable.
