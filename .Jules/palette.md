## 2024-05-20 - Interactive Blockchain Submission Flow
**Learning:** Providing immediate visual feedback for blockchain transactions (loading state -> success message with TxHash) significantly improves perceived reliability and transparency in decentralized application interfaces.
**Action:** Always implement a multi-stage feedback loop for asynchronous actions, including intermediate 'submitting' states and detailed, informative success confirmations (e.g., block numbers, transaction hashes).
