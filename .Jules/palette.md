# Palette's Journal - Critical UX & Accessibility Learnings

## 2025-05-15 - Improving `submit.html` Accessibility and Feedback

**Learning:** Consistent navigation and clear feedback are essential for trust in decentralized applications. Using the document root for dark mode toggling and ensuring all interactive elements are accessible to keyboard users improves the overall user experience. Combining multiple small improvements can sometimes exceed the 'micro-UX' scope but significantly raises the overall quality.

**Action:** Always link logos to the home page, use dual-icon dark mode toggles, and provide visual feedback for long-running (simulated blockchain) actions. Avoid using `alert()` for simulations to maintain a high-quality, non-intrusive UX.
